**Примеры корректных строк:**
1. `auto calc{ [](int a, int b, int c) { return a + (b * c); } };`

**Перечень допустимых лексем:**

* Идентификаторы: `calc`, `a`, `b`, `c`, `x` и т.д.
* Числа: `1`, `2` и т.д.
* Операторы: `=`, `+`, `-`, `*`, `/`
* Спецсимволы: `{`, `}`, `[`, `]`, `(`, `)`, `,`, `;`
* Разделители: пробелы, табы, переносы строк.

## Разработка грамматики G[Z]
Разработанная контекстно-свободная грамматика (РБНФ):

1. `Program` $\rightarrow$ `auto` `id` `Init`
2. `Init` $\rightarrow$ `=` `Lambda` `;` | `{` `Lambda` `}` `;`
3. `Lambda` $\rightarrow$ `[` `]` `(` `ArgList` `)` `{` `return` `Expression` `;` `}`
4. `ArgList` $\rightarrow$ `Type` `id` `ArgListTail` | $\epsilon$
5. `ArgListTail` $\rightarrow$ `,` `Type` `id` `ArgListTail` | $\epsilon$
6. `Type` $\rightarrow$ `int` | `float`
7. `Expression` $\rightarrow$ `Term` `ExpTail`
8. `ExpTail` $\rightarrow$ `AddOp` `Term` `ExpTail` | $\epsilon$
9. `AddOp` $\rightarrow$ `+` | `-`
10. `Term` $\rightarrow$ `Factor` `TermTail`
11. `TermTail` $\rightarrow$ `MultOp` `Factor` `TermTail` | $\epsilon$
12. `MultOp` $\rightarrow$ `*` | `/`
13. `Factor` $\rightarrow$ `id` | `num` | `(` `Expression` `)`

## Классификация грамматики
По классификации Хомского данная грамматика относится к типу 2 (Контекстно-свободные грамматики).

## Метод анализа
Алгоритм синтаксического анализа — **метод рекурсивного спуска**. Для каждого нетерминала грамматики реализована отдельная функция. Решения о выборе правила (в правилах с ветвлением) принимаются по первому символу (Lookahead = 1), поэтому грамматика удовлетворяет условию LL(1).

## Диагностика и нейтрализация синтаксических ошибок (метод Айронса)
Алгоритм работает следующим образом:
1. Если текущий токен не соответствует ожидаемому (ошибка `match`), анализатор формирует сообщение об ошибке (например, "Ожидалось ';'").
2. Начинается фаза **нейтрализации**. Анализатор смотрит на так называемое множество синхронизирующих символов `sync_set` (символов, которые могут следовать за текущей конструкцией).
3. Токены пропускаются (удаляются из входного потока) до тех пор, пока текущий токен не совпадет с одним из символов из `sync_set`. 
4. После обнаружения синхронизирующего символа анализ продолжается со следующего правила без остановки всей программы.

## Тестовые примеры для проверки

Подставьте эти примеры в редактор кода и нажмите клавишу **F5** (или кнопку Пуск). Если код корректный, то над нижней таблицей появится сообщение зеленым цветом: "Ошибок не найдено. Анализ успешно завершен.". В противном случае появится таблица со списком ошибок и общим количеством ошибок. 

### Тест 1: Корректная строка (Инициализация через =)
```cpp
auto func = [](int x, float y) { return x * y + 10; };
```
*Ожидаемый результат: Ошибок не найдено.*

### Тест 2: Корректная строка (Инициализация через {})
```cpp
auto calc{ [](int a, int b) { 
   return a - b / 2; 
} };
```
*Ожидаемый результат: Ошибок не найдено.*

### Тест 3: Пропуск скобок и операторов (Стресс-тест Айронса)
```cpp
auto errorFunc = ](int a int b) { return a + ; };
```
*Ожидаемый результат:* 
Здесь несколько ошибок:
1. Столбец 18 (символ `]`): Ожидалось `[`.
2. Столбец 24 (символ `int` внутри скобок): Ожидалось `,`, так как аргументы перечисляются через запятую.
3. Столбец 44 (символ `;`): Ожидался операнд (ид/число/скобка) после плюса. 
*Обратите внимание, как алгоритм Айронса находит все эти ошибки за один проход, скипая мусор до ближайших синхронизирующих токенов.*

### Тест 4: Целая серия ошибок (Каскад)
```cpp
auto x = []( { return ; } ;
```
*Ожидаемый результат:* Множество ошибок. Пропущены аргументы, скобки, выражения. Алгоритм не падает, а пытается синхронизироваться по скобкам `}` и `;`. В таблице вы увидите более 3 ошибок.

### Тест 5: Полностью пусто или только мусор
```cpp
123 ;
```
*Ожидаемый результат:* Программа выдаст ошибку "Ожидалось 'auto'" и прекратит выполнение, так как `auto` - ключевой стартовый символ.
