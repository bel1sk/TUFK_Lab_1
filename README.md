## Постановка задачи
Программная реализация сканера для выделения лексем из входного текста и интеграция в GUI языкового процессора.

## Вариант задания
**85.** Лямбда-выражения языка C++ (возвращающие арифметическое выражение). Пример: `auto calc{ [](int a, int b, int c) { return a + (b * c); } };`

## Диаграмма состояний (Теоретическая модель FSM)
Конечный автомат лексического анализатора:
* **`START` (Начальное состояние)**: читает текущий символ. По типу символа переходит в одно из следующих состояний. Игнорирует разделяющие пробелы/табы (через вспомогательное состояние `SPACE`).
* **`NUMBER` (Сборка числа)**: читает цифры до появления не-цифры. Фиксирует лексему цифры и возвращается в `START`.
* **`IDENTIFIER` (Сборка слова)**: читает буквы, цифры и подчеркивания. По завершении проверяет собранное слово по таблице ключевых слов (`auto`, `int`, `float`, `return`). Привязывает соответствующий код и возвращается в `START`.
* **Одиночные символы**:
  * Для операторов (`+`, `-`, `*`, `/`, `=`) переходит в соответствующие состояния обработки и сразу возвращается в `START` с кодом 4.
  * Для спецсимволов (`[`, `]`, `(`, `)`, `{`, `}`, `,`, `;`) переходит в уникальные состояния и возвращается в `START` с кодами от 5 до 12.
* **`ERROR`**: фиксируется если символ не распознан.

## Описание интерфейса и функций (руководство пользователя)
* **Работа с файлами**: Поддерживается создание, открытие, сохранение файлов. Возможен Drag-and-Drop.
* **Таблица токенов**: По кнопке "Пуск" (F5) код разбивается на лексемы. Заполняется вкладка "Результаты" (Окно внизу).
* **Таблица ошибок**: Вторая нижняя вкладка служит для отображения некорректных лексем. 
* **Навигация по коду**: При клике на строку в любой из двух нижних таблиц (включая ошибки) исходный текст в верхнем окне подсвечивается в том месте, где находится лексема.
